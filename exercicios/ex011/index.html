Imagen dinâmicas

O uso de imgs dinamicas permite q seu site seja vizualizado rm vários dispositivos diferentes. Usaremos agora a tag <picture>.
Nesse exercício, são usadas três imgs, de 300px, de 700px, e de 1000px

Dentro dessa tag, criaremos outra, <source:media:type>: 

            
                <source media="(max-width: 1020px)" 
                srcset="LINK DA IMAGEM"
                type="image/png">

Audios e podcasts

Agora para audio é até bem parecido. Como eu n quero baixar um audio, tirei print da tela dele e vou escrever o que fazer.

Basicamente, usa-se a tag <audio preload = "metadata" autoplay controls> para definir que ele n dele ser todod carregado, que o play é automático, e que se deve mostrar os controles e tempo de audio, respectivamente.

Usa-se o <source src = "CAMINHO DO AUDIO" type = "audio/TIPO DO AUDIO"> para dizer quais tipos de audio estão deisponíveis, caso dispositivos que suportam diferentes tipos usem o site. A preferencia é de cima para baixo.

Vídeos

Novamente n vou baixar um vídeo, me mama. Assim como audios, é melhor q ele tenha vários tipos para suporte a vários navegadores

Essa é a aula 34, onde ele fala bastante do HandBrake, que pode decodificar em outro tipo de arquivo e também diminuir o tamanho dele.
Não hospede um vídeo muito grande no seu próprio site, pesa e é caro, ou seja, é custoso de todas as formas

Vídeos hospedados localmente

Da mesma forma que audios, usamos:
<video width = "500" poster = "CAMINHO PARA A CAPA DO VIDEO" controls>, com width sendo o tamanho do vídeo no site e poster uma capa não obrigatória que vc queria colocar
<source src = "CAMINHO DO VIDEO" type = "video/TIPO DO AUDIO">

Podemos, q nem com os audios, oferecer várias opções, com prioridade de cima para baixo.
